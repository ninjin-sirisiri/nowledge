# Debug Solid Skill

SolidJSおよびSolidStart特有の問題を解決するためのガイドラインです。

## よくある問題と解決策

### 1. Reactivityが動作しない
値などを更新しても画面が変わらない場合。
- **原因**: Signalのgetterを関数として呼び出していない（例: `count` vs `count()`）。
- **原因**: 分解代入（Destructuring）でPropsを受け取っている（例: `const { title } = props`）。SolidではPropsのReactivityが失われます。
  - **修正**: `props.title` として直接参照するか、`splitProps` または `mergeProps` を使用してください。

### 2. Hydration Mismatch
サーバーサイドレンダリング(SSR)とクライアントサイドでの初回レンダリング結果が一致しないエラー。
- **原因**: `Math.random()` や `Date.now()` など、SSRとクライアントで異なる値を生成する処理をレンダリング中に直接呼んでいる。
- **原因**: ブラウザ固有のAPI（`window`, `document`）をガードなしで使用している。
  - **修正**: `<ClientOnly>` コンポーネントでラップするか、`onMount` フック内で実行するように変更してください。

### 3. APIルートの問題
- **確認**: サーバーコンソールを確認してください。クライアントのエラーログだけでは不十分です。
- **確認**: ネットワークタブでレスポンスヘッダーとボディを確認してください。

## デバッグ手順

1. **ログ出力**: `console.log` を使用する場合、出力先がサーバーターミナルかブラウザコンソールかを意識してください。サーバー関数 (`server$`) 内のログはサーバー側に出力されます。
2. **DevTools**: Solid DevTools拡張機能を使用している場合、Signalの依存グラフを確認して、更新が伝播しているか追跡してください。
3. **最小構成**: 問題を切り分けるため、関係のないコンポーネントを一時的にコメントアウトして再現性を確認してください。
