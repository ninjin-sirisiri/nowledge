# Update Docs Skill

コードの変更はドキュメントの更新とセットで行う必要があります。このスキルは更新漏れを防ぐためのチェックリストです。

## 更新対象となるドキュメント

### 1. `AGENTS.md`
プロジェクトの全体像やコンテキストファイルです。
- **いつ更新するか**: 
  - プロジェクトの構成（Tech Stack）が変わったとき
  - 大きなディレクトリ構造の変更があったとき
  - 新しい「ルール」や「スキル」を追加したとき

### 2. `README.md`
プロジェクトの顔となるファイルです。
- **いつ更新するか**:
  - セットアップ手順が変わったとき
  - 主要な機能が追加/削除されたとき
  - 環境変数の追加が必要になったとき

### 3. `.agent/rules/*.md`
具体的なルールファイル群です。
- **いつ更新するか**:
  - コーディング規約、命名規則を変えたとき
  - アーキテクチャの方針を変更したとき

### 4. JSDoc / コメント
コード内のドキュメントです。
- **いつ更新するか**:
  - 関数の引数や戻り値が変わったとき
  - コンポーネントのPropsが変わったとき
  - 複雑なロジックを変更したとき

## 手順

1. コードの修正完了後、影響範囲を特定します。
2. 上記リストを順に確認し、陳腐化した情報がないかチェックします。
3. 修正が必要な場合、コードのコミットと同じタイミング、あるいは直後のコミットで修正します。
4. `AGENTS.md` のようなAI参照用ファイルは、AIが常に最新の情報を得られるよう、優先的に更新してください。
