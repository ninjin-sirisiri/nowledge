# UI Polish Skill

ユーザーを「WOW」と言わせる、プレミアムでモダンな体験を提供するための最終仕上げプロセスです。

## チェックリスト

### 1. レスポンシブデザイン
- [ ] **Mobile**: 375px幅でレイアウトが崩れていないか。
- [ ] **Tablet**: 768px幅での表示は適切か。
- [ ] **Desktop**: 大画面（1440px+）で間延びしていないか（最大幅の制御）。

### 2. インタラクション & アニメーション
- [ ] **Hover**: クリック可能な要素（ボタン、リンク、カード）すべてにホバーエフェクト（色変化、スケール、シャドウ）があるか。
- [ ] **Feedback**: クリック時にアクティブ状態の変化やリップルエフェクトなどのフィードバックがあるか。
- [ ] **Transitions**: 状態変化（開閉、タブ切り替え）は瞬時に切り替わらず、滑らかなトランジション（CSS transition/animation）がついているか。
- [ ] **Loading**: 読み込み中にスケルトンスクリーンやスピナーが表示されているか。

### 3. ビジュアルデザイン
- [ ] **Spacing**: 余白（margin/padding）は一貫しているか。詰めすぎていないか（十分なホワイトスペース）。
- [ ] **Typography**: フォント階層（H1, H2, Body）は明確か。読みやすいコントラスト比があるか。
- [ ] **Colors**: モダンなカラーパレットを使用しているか。デフォルトの青や赤をそのまま使っていないか。
- [ ] **Glassmorphism**: 適切な場所（ヘッダー、モーダル）に背景のぼかし（backdrop-filter）などのモダンな表現を取り入れているか。

## 実装のヒント

- **Tailwind CSS**: `hover:scale-105`, `transition-all`, `duration-300`, `active:scale-95` などのユーティリティクラスを積極的に活用する。
- **Motion**: 複雑なアニメーションには `Motion One` や `Solid Transition Group` の使用を検討する。
