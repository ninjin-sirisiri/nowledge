# Find Feature Skill

プロジェクト内の特定の機能に関連するコードの場所を特定するためのガイドラインです。Feature-based Architectureを採用しているため、関連ファイルは機能ごとにまとまっています。

## 検索手順

### 1. 機能ディレクトリの確認
まず、`src/features/` ディレクトリ配下を確認します。
- 機能名が明確な場合: `src/features/<feature-name>/` を直接参照します。
- 機能名が不明確な場合: `src/features/` 内のディレクトリリストを参照し、最も近い名前を探します。

### 2. コンポーネントツリーからの逆引き
画面上のUI要素からコードを探す場合:
1. ブラウザのDevToolsや画面上のテキストから、ユニークなクラス名やテキストを特定します。
2. `grep_search` ツールを使用して、その文字列を検索します。
   - 例: ボタンのテキスト "投稿する" で検索
3. ヒットしたコンポーネントファイル (`.tsx`) の場所を確認します。
   - `src/features/` 配下にあれば、それが該当機能のディレクトリです。
   - `src/components/ui/` 配下（共通コンポーネント）の場合、そのコンポーネントを使用している親コンポーネントを検索します。

### 3. ルーティングからの特定
URLからコードを探す場合:
1. `src/routes/` ディレクトリを確認し、URL構造に対応するファイル (`page.tsx` や `layout.tsx`) を見つけます。
2. そのファイル内で import されているコンポーネントを確認します。
   - import パスが `~/features/...` となっていれば、そこが機能の実体です。

## 関連ファイルの特定

機能ディレクトリ (`src/features/<name>/`) が見つかったら、以下の構造で関連ファイルを探せます：

- **UI**: `components/` フォルダ
- **データ取得・送信**: `api/` フォルダ
- **データ型**: `types/` フォルダ
- **状態管理/ロジック**: `hooks/` または `stores/` フォルダ
